当前选型说明：

- P2 初期先采用 `appData/storage/*.json` 的轻量 repository，避免 `better-sqlite3` 等 native 模块影响 Electron 基线和 GitHub Actions 打包。
- repository 层保留清晰边界，后续如果凭证、HTTP 历史、AI 会话规模增长，再迁移到 SQLite 或其他本地数据库。
- 命令管理已支持分组、新增、编辑、删除、搜索、复制；旧项目的页签语义先映射为命令分组。
- 凭证管理已支持新增、编辑、删除、搜索、显示/隐藏、复制账号和复制密码；旧项目批量导入与拖拽排序后续并入“旧数据导入助手 / 备份恢复”。
- Prompt 模板已支持分类、新增、编辑、删除、收藏、变量解析、变量填充和复制结果；导入导出后续统一并入“备份 / 恢复”，避免复刻旧项目总备份不包含 Prompt 的割裂设计。
- 节点列表已补齐 Electron main / preload / IPC 合约，Main Process 通过 `appData/storage/nodes.json` 管理本地节点库；当前先覆盖节点资料管理最小闭环，不在这一轮引入订阅拉取、转换或复杂协议解析。
- 新仓统一备份当前覆盖 `commands + credentials + prompts + nodes + httpCollections`，恢复时按所选模块覆盖写回；节点和 HTTP 集合会按备份里的数组整体恢复到本地资料库。
- 旧数据导入助手已支持识别旧项目总备份 JSON 和旧 Prompt 模板导出 JSON；前者可按模块覆盖迁移命令/凭证/节点，后者按分类名称与模板标题做合并导入。
