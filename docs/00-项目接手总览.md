# doggy-toolbox-web 项目接手总览

## 1. 这个项目现在是什么

`doggy-toolbox-web` 是旧项目 `doggy-toolbox` 的 Electron 重构版。

旧项目是：

```text
Python + PyWebView + 原生 HTML/CSS/JS
```

新项目是：

```text
Electron + electron-vite + Vue 3 + TypeScript + Naive UI
```

这次重构的核心变化不是“把旧仓目录照搬一遍”，而是把职责重新拆层：

- Electron Main Process 负责本地文件、AI SDK、HTTP 请求、本机运行时检测、统一 IPC 入口。
- Preload 负责把可调用能力以 `window.doggy` 的形式安全暴露给前端。
- Renderer 负责页面、交互、状态管理和组件渲染。
- Shared 负责 Main / Preload / Renderer 共用的类型和 IPC 合约。

## 2. 当前已经落地的能力

当前新仓已经完成的主线能力：

- Electron 桌面壳、Vue 3 + TypeScript 工程骨架。
- Naive UI 基础布局、左侧导航、浅色/深色切换。
- 旧项目 9 套主题名称已迁移到新仓外观系统。
- 毛玻璃、透明度、UI 缩放和标题栏视觉模式已在顶部外观条中可配置。
- 本机运行时检测。
- 本地 JSON repository 持久化。
- 命令管理。
- 凭证管理。
- Prompt 模板库。
- 节点列表与分享链接导入。
- HTTP 集合、请求编辑器、环境变量、真实请求发送、请求历史、批量执行。
- cURL / HTTPie / Postman / OpenAPI / Apifox 格式转换。
- 统一备份恢复。
- 旧数据导入助手。
- AI SDK Bridge。
- 工具页 AI 结果复核。

AI 不是通过前端直接发 HTTPS 请求，而是走本机 SDK：

```text
Renderer
  -> preload
  -> ipcMain
  -> AiBridgeService
  -> AiSessionService
  -> AiProviderRouter
  -> CodexSdkBridge / ClaudeAgentBridge
  -> 本机 Codex / Claude 配置与登录态
```

## 3. 当前还没有做完的地方

这点非常重要。当前新仓不是“旧项目 100% 全量功能完成版”。

还没完成的重点主要有两类：

### 3.1 旧工具页没有全量迁移

旧项目 `web/pages` 一共有 54 个页面片段，新仓目前只完成了其中一部分能力。

新仓当前已有页面路由：

- `/`
- `/tools`
- `/commands`
- `/credentials`
- `/prompts`
- `/nodes`
- `/http`
- `/backup`
- `/legacy-import`
- `/ai`
- `/plan`

已经进入统一工作台、但还不是旧项目独立页面级完整迁移的典型工具包括：

- `tool-jwt`
- `tool-hmac`
- `tool-password`
- `tool-regex`
- `tool-sql`
- `tool-csv`
- `tool-markdown`
- `tool-jsonpath`
- `tool-json-schema`
- `tool-ip`
- `tool-cron`
- `tool-mask`
- `tool-toml`
- `tool-datecalc`
- `tool-ua`

仍未形成完整可用入口或仍需独立高级页面的典型工具包括：

- `tool-rsa`
- `tool-crypto`
- `tool-websocket`
- `tool-qrcode`
- `tool-nginx`
- `tool-docker`
- `tool-docker-service`
- `tool-docker-swarm`
- `tool-git`
- `tool-img-base64`
- `tool-diff`
- `tool-mock`

### 3.2 外观 / 主题系统的当前边界

旧项目有一整套“桌面外观设置系统”。新仓当前已经把核心配置项迁到 Renderer 外观 store，并能在界面即时切换。

已迁移：

- 9 套主题：
  `light`、`cute`、`office`、`neon-light`、`cyberpunk-light`、`dark`、`neon`、`cyberpunk`、`void`
- 毛玻璃开关 `glassMode`
- 毛玻璃透明度 `glassOpacity`
- UI 缩放 `uiScale`
- 标题栏模式 `titlebarMode`

尚未一比一复刻：

- 旧项目设置弹窗的“预览草稿 / 保存 / 取消恢复”流程。
- Electron `BrowserWindow` 级别的标题栏参数动态重建。

所以如果你接手这个项目，不要把“当前可运行”误认为“旧项目所有功能都已经迁完”。

## 5. 代码入口速查

### 5.1 启动入口

- Electron Main 入口：`src/main/index.ts`
- Preload 入口：`src/preload/index.ts`
- Renderer 入口：`src/renderer/src/main.ts`
- 路由定义：`src/renderer/src/router/index.ts`
- IPC 合约：`src/shared/ipc-contract.ts`

### 5.2 关键分层

- Main 服务层：`src/main/services/*`
- Renderer 状态层：`src/renderer/src/stores/*`
- Renderer 业务页面：`src/renderer/src/features/*`
- 页面壳：`src/renderer/src/components/AppShell.vue`

## 6. 用一句话概括当前工程状态

当前项目已经具备“可运行、可打包、可继续开发”的现代 Electron 基线，也已经完成 AI SDK Bridge 和多块本地工具能力，但距离“旧项目全量功能与外观完整迁移”还有明确差距，需要继续按模块推进。
