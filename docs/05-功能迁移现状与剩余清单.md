# 功能迁移现状与剩余清单

## 1. 先说结论

当前新仓不是旧项目功能全量完成版。

更准确的状态是：

- 现代 Electron 工程骨架已完成。
- 本地资料类模块已形成新仓主实现。
- HTTP 集合、AI SDK Bridge、备份恢复、旧数据导入已经能真实使用。
- 旧项目工具已收敛到统一工具工作台，当前共 43 个入口，其中 12 个是高级面板。
- 旧项目全局搜索体验已迁成 Vue 版，支持搜索、收藏、最近使用、使用次数统计和工具直达。
- AI 设置已经落地为 SDK-only 持久化配置，并被 AI 主对话、工具、命令、Prompt、节点、HTTP 页面消费。
- 外观核心配置已经迁移到“顶部快捷条 + 设置弹窗预览/保存/取消恢复”，但仍属于轻量替代，不是窗口级完整 titlebar 重建。
- 旧项目大量独立工具页仍未完成页面级迁移。

后续判断迁移状态时，不再只用“完成 / 未完成”二分，而是统一分成三类：

- **已对齐**：新仓已有真实可用入口、数据链路或纯函数闭环，可作为当前主实现继续维护。
- **轻量替代**：核心能力已有，但 UI 形态、交互深度、旧页面细节或高级状态没有一比一迁移。
- **未完成**：旧项目能力还没有形成新仓可用入口，或仍缺独立高级 UI / 持久化 / 测试闭环。

## 2. 旧项目与新项目的范围差异

### 2.1 旧项目

旧项目：

- `web/index.html` 中包含大量 `data-page` 导航入口。
- `web/pages` 下共有 54 个页面片段。
- 很多工具是独立 HTML 片段 + 原生 JS 交互。
- AI 路线是 `pywebview.api -> Python api.py -> services/ai_providers.py -> 手写 HTTPS`。

### 2.2 新项目

新项目当前主线页面是：

- 总览
- 工具
- 命令管理
- 凭证管理
- Prompt 模板
- 节点列表
- HTTP 集合
- 备份恢复
- 旧数据导入
- AI Bridge
- 排期

新仓不是把旧 `web/pages` 逐个改成 Vue 页面，而是先把高价值能力收敛到更清晰的模块：

- 本地资料类模块统一走 Main Service + JSON repository。
- 纯前端工具统一进入工具工作台。
- 网络请求统一进入 HTTP 集合。
- AI 统一进入本机 SDK Bridge。
- 页面级 AI 入口统一受 `AiSettingsService` 的全局开关和模块开关控制。
- 外观设置统一进入 Renderer 外观 store、顶部快捷条和设置弹窗。

## 3. 已对齐能力

这些能力已经能作为新仓当前主实现继续维护。

### 3.1 工程基线

- Electron Vite 工程。
- Vue 3 + TypeScript。
- Naive UI。
- GitHub Actions 打包。
- 多平台 electron-builder 配置。
- `npm run test`、`npm run typecheck`、`npm run lint`、`npm run build` 这套基础验证脚本。

### 3.2 本地资料类模块

- 命令管理：分组、命令 CRUD、搜索、复制、本地落盘。
- 凭证管理：新增、编辑、删除、搜索、显示/隐藏、复制账号/密码，支持 `safeStorage` 可用时编码敏感字段。
- Prompt 模板：分类、模板、变量解析、变量填充、收藏、复制结果。
- 节点列表：本地节点资料库、标签筛选、分享链接导入。
- HTTP 集合：集合、请求、环境变量、真实请求发送、历史记录、批量测试。
- 统一备份恢复：覆盖已迁移模块的数据导出和按模块覆盖恢复。
- 旧数据导入：支持旧项目总备份 JSON 与旧 Prompt 模板导出 JSON。

### 3.3 AI SDK Bridge

- 本机 Codex 配置检测。
- 本机 Claude 配置检测。
- Codex SDK Bridge。
- Claude Agent SDK Bridge。
- AI 设置持久化：默认工作目录、系统提示、全局开关、模块级开关。
- 统一 AI 流式事件协议。
- AI 历史会话保存。
- runtime / thinking / tool / usage / provider session 元数据保留。
- 工具页 AI 结果复核。
- 命令页 AI 说明、Prompt 页 AI 审查、节点页 AI 说明、HTTP 页 AI 分析。
- `npm run smoke:ai` 真实本机 SDK 冒烟入口。

### 3.4 首批纯前端工具

这些工具在新仓统一工作台里已经比较接近主实现状态：

- Base64
- URL
- UUID
- Hash
- Time
- JSON
- Text
- Unicode
- Radix
- Naming

## 4. 轻量替代能力

这些能力已经可用，但不能说旧项目页面级完整迁移。

### 4.1 旧工具统一工作台入口

下面这些旧项目工具已经迁入统一工具工作台，优先覆盖纯函数和轻交互：

- JWT
- HMAC
- HTML Entity
- Char Count
- Text Sort
- Data Mask
- Regex
- SQL
- CSV
- Markdown
- Color
- Cron
- Password
- JSON Schema
- JSONPath
- TOML
- User Agent
- IP
- Base64 / Hex
- Date Calc
- Data Convert

当前边界：

- 已迁移的是算法能力或统一 textarea 工作台入口。
- 多数没有复刻旧项目独立页面布局。
- 复杂表单、文件拖拽、连接状态、图片预览、密钥生命周期等高级交互仍未补齐。

### 4.1B 全局搜索与工具直达

已完成：

- `⌘K / Ctrl+K` 打开全局搜索弹窗。
- 搜索模块和 43 个工具入口。
- 工具收藏。
- 最近使用。
- 使用次数统计排序。
- `/tools?tool=xxx` 直达指定工具。

当前边界：

- 这是旧项目全局搜索的 Vue 版轻量迁移，数据仍存 `localStorage`。
- 不把收藏和使用次数纳入统一备份。
- 不复刻旧项目侧边栏 DOM 反查机制，改为消费 TypeScript 注册表。

### 4.1A 已接入高级面板但仍属轻量替代的工具

- QR Code
- Image Base64
- Mock
- Diff
- Crypto
- RSA
- WebSocket
- Git
- Docker
- Docker Service
- Docker Swarm
- Nginx

当前边界：

- 这些工具都已经形成新仓可用入口，并能回传 snapshot 给工具工作台和 AI 复核。
- 但它们仍是统一工作台中的高级面板，不是旧项目那种独立页面级完整迁移。
- 复杂拖拽、更多导出格式、更多持久化配置和更强校验策略仍有继续补齐空间。

### 4.2 HTTP 格式转换

已完成：

- cURL 导入。
- cURL 导出。
- HTTPie 导出。
- Postman Collection v2.1 导入导出。
- OpenAPI / Apifox 基础 JSON 导入导出。

当前边界：

- 当前统一收敛到 HTTP 集合工作台。
- 暂不支持 YAML 解析。
- 暂不做复杂 `$ref` 全量展开。
- 不等于旧项目所有网络类工具页面一比一还原。

### 4.3 节点转换

已完成：

- 本地节点资料库。
- 标签筛选。
- `vmess://`、`vless://`、`trojan://`、`ss://` 分享链接导入。

当前边界：

- 订阅拉取未完成。
- 更复杂协议解析未完成。
- 更完整的节点转换生态未完成。

### 4.4 外观系统

已完成：

- 9 套主题：
  `light`、`cute`、`office`、`neon-light`、`cyberpunk-light`、`dark`、`neon`、`cyberpunk`、`void`
- 顶部外观条。
- 外观设置弹窗。
- 毛玻璃开关。
- 毛玻璃透明度滑块。
- UI 缩放滑块。
- 标题栏 `fixed / minimal` 视觉模式。
- 设置弹窗预览草稿 / 保存 / 取消恢复。
- `localStorage` 持久化。

当前边界：

- 这是旧项目外观设置的核心体验迁移，但仍属于 renderer 级外观方案。
- 标题栏模式当前是 Renderer 视觉模式，不动态重建 Electron `BrowserWindow` 参数。
- 没有逐像素复刻旧项目所有组件主题细节。

## 5. 未完成能力

下面这些能力当前仍未形成完整迁移闭环。

### 5.1 高级工具的独立页面级完整体验

- Mock / Diff / Crypto / Git / Docker / Docker Service / Docker Swarm / Nginx 当前都已接入统一工作台高级面板。
- 但这些工具还没有拆成旧项目那种独立 Vue 页面，也没有把所有深交互、复杂导出和更强配置管理全部补齐。
- QR Code、Image Base64、RSA、WebSocket 也是同样情况：当前可用，但仍属于轻量替代。

### 5.2 旧 AI 设置体验

- 旧项目 `ai-settings.html` 的 HTTPS Provider CRUD 未迁移。
- 新仓已改为 SDK-only AI 设置页，并已接入 `AiChatView`。
- 默认工作目录、system prompt、全局开关、模块级开关已经落到本地资料库 `storage/ai-settings.json`。
- 页面级 AI 能力已覆盖 AI 主对话、工具、命令、Prompt、节点和 HTTP 集合。
- 更复杂多轮会话管理仍未完成。

### 5.3 旧外观设置体验

- Electron 窗口级标题栏动态切换未完成。
- 更细粒度的组件级主题差异和更完整的窗口系统集成未完成。

### 5.4 旧工具页导航形态

- 旧项目 50+ 页面还没有全部变成独立 Vue 页面。
- 新仓当前优先使用统一工作台和模块化页面。
- 后续如果要追求旧项目“每类工具一个独立页面”的形态，需要按优先级继续拆。

## 7. 后续建议排期

如果继续推进，建议按下面顺序做：

### P7-A 高频高级工具页面

- 补二维码 PNG 导出和更完整下载选项。
- 补图片 Base64 拖拽预览。
- 补 WebSocket 更多连接配置。
- 补 RSA 更多算法族或独立管理体验。

### P7-B 成体系命令生成器

- Docker 独立页面级完整体验。
- Docker Service 独立页面级完整体验。
- Docker Swarm 独立页面级完整体验。
- Git 独立页面级完整体验。
- Nginx 独立页面级完整体验。

### P7-C 旧体验补齐

- 更复杂多轮 AI 会话管理。
- 外观系统与 Electron `BrowserWindow` 标题栏参数的更深集成。
- 全局搜索收藏 / 最近使用是否需要纳入统一备份。
- 旧项目每类工具独立 Vue 页面级体验。

### P8 接手性补强

- Renderer 大型业务组件继续分批补注释。
- 把旧项目功能矩阵继续沉淀到 `重构功能待办.md`。
- 每补一个独立页面，就同步更新本文和 `04-模块使用说明与按钮调用链.md`。

## 8. 接手时不要误判的几点

- `重构功能待办.md` 里阶段完成，不等于旧项目所有页面完成。
- 当前测试通过，说明“现有新仓能力稳定”，不代表“旧功能全量到位”。
- AI 已经是真实可用，不是 mock；`ai-settings` 已改成 SDK-only 真实持久化，不迁移旧 HTTPS Provider CRUD。
- 主题系统核心配置和设置弹窗草稿流已可用，但仍属于 renderer 级轻量迁移。
- 工具工作台有入口，不等于旧项目对应工具页已经独立页面级完成。
