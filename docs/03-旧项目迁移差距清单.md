# 旧项目迁移差距清单（索引）

这份文件保留为历史入口和跳转索引。迁移差距的详细台账已经统一维护在：

- `05-功能迁移现状与剩余清单.md`
- `重构功能待办.md`

## 1. 为什么不在这里继续写完整清单

旧迁移差距会随着 P3 / P4 / P6 / P7 持续变化。

如果本文件继续保留完整正文，就会和 `05-功能迁移现状与剩余清单.md`、`重构功能待办.md` 形成三份状态源，后续很容易出现“一个文件说已完成，另一个文件说未完成”的口径冲突。

所以当前规则是：

- 本文件只做索引。
- 迁移状态以 `05-功能迁移现状与剩余清单.md` 为准。
- 执行进度、阶段勾选和执行日志以 `重构功能待办.md` 为准。

## 2. 当前统一分类

详细内容请看 `05-功能迁移现状与剩余清单.md`，这里只保留分类口径：

- **已对齐**：在新仓已有真实页面或模块闭环，并能通过 Main / Preload / Renderer 或纯前端工具链实际使用。
- **轻量替代**：核心算法、配置项或基础操作已有，但 UI 形态、交互深度或旧页面细节没有一比一复刻。
- **未完成**：旧项目能力在新仓尚未形成可用入口，或仍缺独立高级 UI / 持久化 / 测试验证。

## 3. 当前不要误判

- 统一工具工作台里的工具入口不等于旧项目对应页面完整迁移。
- 主题、毛玻璃、缩放、标题栏视觉模式已可用，但不等于旧项目设置弹窗完整迁移。
- HTTP 集合已经是新仓的主力网络工作台，但不等于旧项目所有网络类工具页面都已一比一还原。
- AI SDK Bridge 已可真实工作，但旧项目 `ai-settings` 页面体验仍未完整迁移。
