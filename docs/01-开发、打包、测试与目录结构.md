# 开发、打包、测试与目录结构

## 1. 环境要求

当前仓库主要依赖：

- Node.js 22
- pnpm 10（通过 `packageManager` 锁定，建议使用 Corepack 启用）
- Electron 39
- electron-vite 5

AI 相关本机条件：

- Codex 路径下至少要有 `~/.codex/config.toml` 或 `~/.codex/auth.json`
- Claude 路径下至少要有 `~/.claude.json`
- Codex / Claude SDK 默认不作为主应用依赖安装；AI 设置页会按需把 SDK runtime 安装到用户数据目录。
- 开发态 runtime 安装可回退系统 `pnpm`、`corepack pnpm` 或 `npm`；打包后的 `macOS arm64` / `Windows x64` 应用优先使用随包附带的 runtime installer。

如果本机没有这些配置，项目仍然可以启动，但 AI 页面只会显示“未检测到配置”。

## 2. 常用命令

`package.json` 当前脚本如下：

```json
{
  "format": "prettier --write .",
  "lint": "eslint . --max-warnings=0",
  "typecheck": "vue-tsc --noEmit -p tsconfig.web.json && tsc --noEmit -p tsconfig.node.json",
  "test": "vitest run",
  "smoke:ai": "node scripts/smoke-local-ai-sdk.mjs",
  "dev": "electron-vite dev",
  "build": "pnpm run typecheck && electron-vite build",
  "postinstall": "electron-builder install-app-deps",
  "build:unpack": "pnpm run build && electron-builder --dir",
  "build:mac": "pnpm run build && electron-builder --mac",
  "build:win": "pnpm run build && electron-builder --win",
  "build:linux": "pnpm run build && electron-builder --linux"
}
```

### 2.1 本地开发

```bash
pnpm install
pnpm run dev
```

`pnpm run dev` 会同时启动：

- Electron Main / Preload 构建监听
- Renderer Vite 开发服务
- Electron 桌面窗口

### 2.2 基础校验

```bash
pnpm run test
pnpm run typecheck
pnpm run lint
pnpm run build
```

建议改动后至少跑：

- 改类型或接口：`pnpm run typecheck`
- 改逻辑：`pnpm run test`
- 改页面和逻辑一起：`pnpm run lint && pnpm run typecheck`
- 准备交付前：`pnpm run build`

### 2.3 AI 真实冒烟

```bash
pnpm run smoke:ai
pnpm run smoke:ai -- codex
pnpm run smoke:ai -- claude
```

这不是 mock。它会真实调用已安装的本机 SDK runtime。
如果对应 runtime 还没安装，先到 AI 设置页安装，或使用临时 `DOGGY_AI_RUNTIME_ROOT` 指向已经准备好的 runtime 目录。
Codex 会写 session 文件；如果当前环境不能写 `~/.codex/sessions`，可以用临时 `HOME` 做 smoke 验证。

## 3. GitHub Actions 打包

工作流文件：

- `.github/workflows/build.yml`

当前触发方式有两种：

- 推送 `v*` tag
- 在 GitHub Actions 页面手动触发

当前 CI 做的事情：

1. `corepack enable`
2. `pnpm install --frozen-lockfile`
3. `pnpm run typecheck`
4. `pnpm run lint`
5. `pnpm run build`
6. `pnpm exec electron-builder --publish never --mac --arm64` 或 `pnpm exec electron-builder --publish never --win --x64`
7. 上传构建产物

本仓库已经固定 `packageManager` 为 pnpm，并使用 `pnpm.onlyBuiltDependencies` 只允许 `electron` 和 `esbuild` 执行安装期构建脚本。这样可以避免 npm 平铺安装造成的重复依赖和不必要构建脚本扩散。

```json
{
  "pnpm": {
    "onlyBuiltDependencies": ["electron", "esbuild"]
  }
}
```

本地如果曾经用 npm 安装过依赖，迁移到 pnpm 后可能出现 `node_modules/.ignored` 旧包残留。它不会参与当前 pnpm 依赖解析，但会继续占用磁盘空间；确认迁移后应删除旧残留或重建 `node_modules`。

正式使用 pnpm 后，仓库应只保留 `pnpm-lock.yaml`，不要同时维护 `package-lock.json`，避免 CI 和本地安装来源不一致。

当前 GitHub Actions 只面向两类安装包：

- `macOS arm64`：面向 Apple Silicon / M 芯片 Mac。
- `Windows x64`：面向常见 64 位 Windows。

当前上传的产物包括：

- `release/*.dmg`
- `release/*.zip`
- `release/*.exe`
- `release/*.AppImage`
- `release/*.tar.gz`
- `release/latest*.yml`
- `release/*.blockmap`

## 5. 重点目录解释

### 5.1 `src/main`

主进程目录。

负责：

- 注册所有 `ipcMain.handle(...)`
- 管理 `BrowserWindow`
- 读取本机配置
- 管理本地 JSON repository
- 执行 HTTP 请求
- 调用 Codex / Claude SDK

### 5.2 `src/preload`

安全桥目录。

负责：

- 把可调用 API 暴露到 `window.doggy`
- 隔离 Renderer 与 Electron 原生能力
- 统一 `ipcRenderer.invoke(...)` 和事件监听

### 5.3 `src/renderer/src`

Vue 前端目录。

再往下可以这样分：

- `views/`：路由页面薄壳
- `features/`：真正业务组件
- `stores/`：Pinia 状态管理
- `components/`：应用壳、公共组件
- `router/`：路由定义
- `styles/`：主题样式

### 5.4 `src/shared`

共享类型目录。

核心文件：

- `ipc-contract.ts`

这个文件非常重要，因为它定义了：

- Runtime 信息结构
- AI 事件协议
- 各模块 state 类型
- 各模块 save input 类型
- `BridgeApi` 接口

Main、Preload、Renderer 都依赖它来保持一致。

## 7. 主题现状

当前新仓不是旧项目的完整主题系统。

现在已经完成的是“核心外观配置迁移”：

- 9 套主题：
  `light`、`cute`、`office`、`neon-light`、`cyberpunk-light`、`dark`、`neon`、`cyberpunk`、`void`
- Renderer 外观 store
- 顶部外观条即时切换
- 毛玻璃开关
- 毛玻璃透明度
- UI 缩放
- 标题栏 `fixed / minimal` 视觉模式
- `localStorage` 持久化

还没有一比一复刻旧项目的：

- 设置弹窗的“预览草稿 / 保存 / 取消恢复”流程
- Electron `BrowserWindow` 级别标题栏参数动态重建
- 所有旧页面和旧 CSS 组件状态的像素级主题适配

所以这里的准确表述是：外观核心能力已可用，但仍属于旧外观系统的轻量迁移，不是完整复刻。

## 8. 建议的排障顺序

如果项目跑不起来，建议按下面顺序排查：

1. `pnpm install` 是否成功
2. `pnpm run typecheck` 是否通过
3. `pnpm run dev` 是否正常打开窗口
4. `window.doggy` 是否成功挂载
5. `runtime:get-info` 是否能返回
6. 本机 `~/.codex` / `~/.claude.json` 是否存在
7. `pnpm run smoke:ai` 是否能通过
